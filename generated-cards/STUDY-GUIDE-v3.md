## üß† Learning Techniques

### 1. Active Recall (Core Principle)

**DON'T:**
- ‚ùå Read the question and immediately reveal answer
- ‚ùå Passively recognize correct formula
- ‚ùå Rate "Good" if you peeked

**DO:**
- ‚úÖ Cover answer, speak solution out loud
- ‚úÖ Write calculation steps on paper
- ‚úÖ Only reveal after genuine attempt
- ‚úÖ Rate "Again" if you couldn't explain to someone else

**Why:** Retrieval strengthens memory +40% vs passive review

---

### 2. Elaborative Interrogation

**For every card, ask:**
- "Waarom?" (Why does this formula work?)
- "Wat als..." (What if I change this value?)
- "Hoe herken ik dit?" (How do I recognize this problem type?)

**Example (Card R05):**
```
Question: Motor cos œÜ calculation

Don't just: "cos œÜ = P/S = 60/68 = 0.88"

Also ask yourself:
- Why is cos œÜ always ‚â§ 1? (Definition P/S, P ‚â§ S)
- What if cos œÜ = 0? (Pure reactive, no useful work)
- How would I improve cos œÜ? (Add compensation capacitors)
- What happens to current if I improve cos œÜ? (Decreases!)
```

**Benefit:** Deep understanding, not rote memorization

---

### 3. Interleaving

**DON'T:**
- ‚ùå Study all "vermogen" cards in one session
- ‚ùå Complete one domain before moving to next

**DO:**
- ‚úÖ Mix different domains in each session
- ‚úÖ Use Anki's default algorithm (it interleaves automatically)
- ‚úÖ Create custom filtered decks alternating topics

**Why:** Discrimination learning +25% retention

---

### 4. Spaced Repetition (Anki's Superpower)

**Trust the algorithm:**
- If Anki shows a card "too soon" ‚Üí it knows you're forgetting
- If a card doesn't appear for weeks ‚Üí you've mastered it
- "Again" ratings aren't failure ‚Üí they're data for optimization

**Optimal review schedule:**
- Day 1: Learn card
- Day 3: First review
- Day 7: Second review
- Day 14: Third review
- Day 30: Fourth review
- etc. (exponential intervals)

**Don't:**
- ‚ùå Cram all cards in one day
- ‚ùå Skip reviews (breaks the algorithm)
- ‚ùå Always rate "Easy" (inflates intervals)
